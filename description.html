Изменения

  1. Каллбек .addHExt(master, rc) заменён на .addSecondHeaderLine(master, rc).
  2. Если каллбек .addSecondHeaderLine(master, rc) не задать, тогда соответствующие ряды отображаться не будут. Дерево станет в два компактнее.

Каллбеки

    Обязательные

  1. .getPath(pathArr) - возвращает путь, сформированный из массива имён, используя необходимый разделитель.

    Иногда бывает, что для формирования пути необходимо использовать разные разделители, например obj.a.b[4].c[1].

    pathArr - массив имён всех вышестоящих иерархических пунктов.


  2. .addHeader(master, rc) - Создаёт заголовок (элемент иерархии).

    rc - массив типа [row, cell], определяющий координаты ячейки.


  3. .addBranchEl(type, master, rc) - Создаёт элемент ветки.

    type - тип элемента ветки:

    'v' vertical " ┃ " вертикальная часть стебля
    'f' fork " ┣━" развилка
    'c' corner " ┗━" угол
    'e' empty " " пустая ячейка

    4. Полу-обязательные

  4. .addSecondHeaderLine(master, rc) - Создаёт дополнительную строку под заголовком, относящуюся к заголовку.

    Это бывает необходимо для размещения дополнительной информации или элементов управления.

    -Если этот каллбек задать, то для отображения одного узла отренрерится две строки.
    -Если этот каллбек не задать, то для отображения одного узла отренрерится одна строка. Дерево станет компактнее.


  5. .newRow(master, rI) - Создаёт новую строку.

    Вызывается в начале формирования строки.

    rI - (row index) - номер строки дерева.


  6. .endOfRow(master, rI) - Делает перевод строки.

    Вызывается в конце формирования строки.


    7. Не обязательные

  7. .getNameToPath(master) - генерирует имя узла, которое подставится в массив пути pathArr, который передастся каллбеку .getPath(pathArr).

    Если данный каллбек не задать, то в pathArr подставится значение поля .name мастер-объекта.

    Важно!
    Свойство задаётся через return.

  8. .initNodeBefore(master) - Предворительная фаза инициализации мастер-объекта.

    Бывает, что до начала создания узла дерева мастер-объект нужно довычислить. Это можно сделать здесь.

    master - мастер-объект соответствующего иерархического элемента.


  9. .initNodeAfter(master) - Завершающая фаза инициализации мастер-объекта.

    Срабатывает на пост-рекурсивной фазе, когда уже создан и сам узел и его дети. Бывает полезен для того, чтобы например наделить узел (или детей) ссылкой на мастер-объект.

    master - мастер-объект соответствующего иерархического элемента.
